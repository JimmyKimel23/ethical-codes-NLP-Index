# Методология мультитематического анализа кодексов этики

## 1. Цель

Методика предназначена для воспроизводимого анализа корпоративных кодексов этики из разных стран и индустрий по нескольким тематическим направлениям (`topic_id`).

Базовые этапы:

1. инвентаризация корпуса;
2. анализ каждого кодекса по выбранной теме;
3. синтез унифицированного набора положений по теме.

## 2. Единицы анализа

- Базовая единица документа: один кодекс (`document_id`).
- Базовая единица тематического анализа: пара `document_id × topic_id`.

## 3. Источники правил

- `docs/topic_registry.md` — список тем и их определений.
- `docs/principles/<topic_id>.md` — канонические принципы темы.
- `docs/status_scale.md` — шкала статусов (переиспользуется для всех тем).

## 4. Общие правила анализа

- Анализ ведётся по языку оригинала документа.
- Цитаты сохраняются в оригинале, переводятся на русский.
- Нормализация и выводы фиксируются на русском.
- Нельзя подменять отсутствие текста догадками.

## 5. Категории релевантности

Категории релевантности задаются **на уровне конкретной темы** в `docs/topic_registry.md`.

Для темы `competitor_relations` используются категории:

1. `competition_law_compliance`
2. `anti_collusion`
3. `information_exchange_controls`
4. `competitor_contacts_protocol`
5. `competitor_information_integrity`
6. `fair_competition_and_non_disparagement`
7. `escalation_and_reporting`
8. `other_competitor_related`

Категории используются для типизации фрагментов и не заменяют канонические `principle_code`.

## 6. Порядок анализа одной темы в одном документе

1. Проверить наличие явного раздела по теме.
2. Если явного раздела нет, искать функциональные эквиваленты.
3. Присвоить `topic_status`.
4. Заполнить `key_excerpts` и связать цитаты с `supports_principles`.
5. Сформировать `principle_codes_detected` как объединение `supports_principles`.
6. Обновить `registry/topic_analysis_status.csv`.

## 7. Синтез

Синтез выполняется отдельно для каждой темы (`topic_id`) на основе завершённых карточек этой темы.

Результаты складываются в `synthesis/<topic_id>/`.

## 8. Критерии качества

Результат качественный, если:

1. вывод воспроизводим по цитатам;
2. статусы присвоены по единой шкале;
3. цитаты и интерпретация разделены;
4. используется канонический реестр принципов темы;
5. синтез опирается на карточки, а не на предположения.
