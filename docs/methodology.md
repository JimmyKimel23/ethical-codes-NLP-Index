# Методология анализа кодексов этики по теме «Отношения с конкурентами»

## 1. Цель

Методика предназначена для воспроизводимого анализа корпоративных кодексов этики из разных стран и индустрий с фокусом на тему **«Отношения с конкурентами»**.

Задача состоит из трёх последовательных этапов:

1. инвентаризация корпуса;
2. анализ каждого кодекса;
3. синтез унифицированного набора положений для «идеального» раздела кодекса.

## 2. Принцип атомарности

Каждый этап выполняется отдельно.

- Инвентаризация не подменяет анализ.
- Анализ одного кодекса не подменяет синтез.
- Синтез запускается только после накопления достаточного массива завершённых карточек.

Это снижает риск ложной уверенности и помогает удерживать единые правила классификации.

## 3. Единица анализа

Единицей анализа является **один кодекс** (или его функциональный эквивалент: code of conduct, business conduct guidelines и т.п.).

Для каждого документа создаётся отдельная YAML-карточка в `analysis/yaml/`.

## 4. Правило мультиязычности

- Анализ ведётся по языку оригинала документа.
- Ключевые цитаты сохраняются в оригинале.
- Для каждой ключевой цитаты обязателен перевод на русский.
- Нормализация принципов, статусы и итоговые аналитические выводы записываются на русском.
- Если перевод неоднозначен, это фиксируется в `limitations`, а качество доказательной базы при необходимости понижается.

## 5. Что считается релевантным теме «Отношения с конкурентами»

Используются восемь категорий релевантности:

1. `competition_law_compliance` — соблюдение антимонопольного / конкурентного законодательства.
2. `anti_collusion` — запрет сговора с конкурентами, включая цены, рынки, клиентов, территории, объёмы и тендеры.
3. `information_exchange_controls` — ограничения на обмен чувствительной коммерческой информацией.
4. `competitor_contacts_protocol` — правила контактов с конкурентами, отраслевыми ассоциациями и мероприятиями.
5. `competitor_information_integrity` — запрет незаконного или неэтичного получения/использования информации о конкурентах.
6. `fair_competition_and_non_disparagement` — добросовестная конкуренция, запрет дискредитации и ложных заявлений.
7. `escalation_and_reporting` — консультации с legal/compliance и сообщения о нарушениях или рисках.
8. `other_competitor_related` — иной действительно новый релевантный блок.

Категории используются для типизации найденных фрагментов, но не заменяют реестр канонических принципов.

## 6. Порядок анализа одного кодекса

### Шаг 1. Поиск явного раздела
Сначала проверяется, есть ли раздел, прямо посвящённый отношениям с конкурентами.

### Шаг 2. Поиск функциональных эквивалентов
Если явного раздела нет, ищутся эквиваленты, например:

- fair competition
- antitrust
- competition law
- market conduct
- dealings with competitors
- competitor information
- industry associations

### Шаг 3. Оценка распределённости материала
Если положения разбросаны по нескольким разделам, это фиксируется отдельно.

### Шаг 4. Квалификация статуса
Документ получает один статус наличия темы и один статус качества доказательной базы.

### Шаг 5. Нормализация в закрытый реестр принципов
Найденные положения не формулируются заново каждый раз, а относятся к фиксированным кодам принципов из `docs/principle_registry.md`.

### Шаг 5a. Привязка цитат к принципам
Каждая запись в `key_excerpts` содержит поле `supports_principles`, в котором перечислены `principle_code`, подтверждаемые этой цитатой.

Итоговый `principle_codes_detected` формируется как объединение всех `supports_principles` из карточки.
Это обеспечивает атомарную прослеживаемость: для каждого принципа можно найти конкретную цитату.

## 7. Статусы

Определения статусов вынесены в `docs/status_scale.md` и используются без отклонений.

## 8. Критерий доказательности

Цитата в `key_excerpts.original` ограничена 2-3 предложениями. Если фрагмент длиннее, он разбивается на отдельные записи.

Каждый вывод по документу должен быть подтверждён:

- точным заголовком раздела, если он есть;
- короткой цитатой из оригинала;
- переводом на русский;
- комментарием, почему именно этот фрагмент релевантен теме.

Если таких подтверждений недостаточно, понижается `evidence_quality` и при необходимости ставится `needs_human_review`.

## 9. Принцип отделения текста от интерпретации

Нужно строго различать:

- что **прямо сказано** в кодексе;
- что является **аналитической нормализацией**.

Нельзя подменять отсутствие текста общими догадками.

## 10. Правила для синтеза

Синтез опирается только на завершённые карточки с достаточной доказательной базой.

Уровни синтеза определяются по функции положения:

### Basic
Без этого положения раздел не выполняет регуляторную и поведенческую функцию.

### Strengthening
Положение повышает операционную применимость и помогает сотруднику действовать правильно в реальных ситуациях.

### Advanced
Положение повышает зрелость и детализацию регулирования, но не образует минимально необходимое ядро.

Частота встречаемости учитывается отдельно и не определяет уровень автоматически.

## 11. Критерии качества результата

Результат считается качественным, если одновременно соблюдены условия:

1. по каждому кодексу есть воспроизводимый вывод;
2. статус присвоен по единым правилам;
3. цитаты и нормализация не смешаны;
4. дрейф категорий ограничен закрытым реестром принципов;
5. синтез опирается на корпус, а не на произвольные предположения.
