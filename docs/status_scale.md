# Шкала статусов

## 1. Статус наличия темы: `competitor_relations_status`

Используется ровно одно значение из списка ниже.

### `explicit_section`
Есть отдельный раздел, прямо посвящённый отношениям с конкурентами,  
или раздел с очевидно эквивалентным названием и содержанием.

Примеры:
- Antitrust and Competition
- Dealing with Competitors
- Fair Competition
- Competition Law Compliance

### `implicit_scattered`
Отдельного раздела нет, но релевантные положения есть в нескольких частях кодекса.

Признаки:
- тема раскрыта не в одном блоке;
- положения разбросаны между general compliance, market conduct, confidentiality, reporting и т.п.;
- суммарно эти положения дают содержательную картину.

### `brief_mention`
Тема упоминается кратко, но без развёрнутых правил поведения.

Признаки:
- одна или две общие фразы;
- нет операциональных запретов/обязанностей;
- нет понятных ориентиров для сотрудника.

### `absent`
Релевантные положения по теме не обнаружены.

Используется только если после добросовестного поиска не найдено ни явного раздела, ни функциональных эквивалентов, ни содержательных фрагментов.

### `unclear`
Нельзя уверенно классифицировать документ.

Типичные причины:
- текст неполный;
- низкое качество извлечения;
- неоднозначный перевод;
- отсутствуют страницы;
- документ повреждён или фрагментарен.

---

## 2. Качество доказательной базы: `evidence_quality`

### `high`
Есть прямые и понятные подтверждения:
- явные цитаты;
- ясные формулировки;
- устойчивый перевод;
- логика квалификации не вызывает сомнений.

### `medium`
Подтверждения есть, но они не идеально прямые.

Примеры:
- положения разбросаны по нескольким разделам;
- формулировки косвенные;
- часть интерпретации требует осторожности.

### `low`
Доказательная база слабая.

Примеры:
- документ извлечён не полностью;
- перевод сомнителен;
- структура текста нарушена;
- ключевые места плохо читаются.

---

## 3. Статус обработки: `review_status`

### `not_started`
Документ занесён в реестр, но анализ ещё не выполнялся.

### `in_progress`
Документ находится в работе, карточка ещё не завершена.

### `completed`
Анализ завершён и пригоден для включения в корпус.

### `needs_human_review`
Есть ограничения, которые мешают доверять результату без дополнительной проверки человеком.

---

## 4. Готовность к синтезу: `synthesis_ready`

### `yes`
Карточка завершена и может использоваться для синтеза.

Обычно это означает:
- `review_status = completed`
- `competitor_relations_status != unclear`
- доказательная база достаточна для сравнительного анализа

### `no`
Документ пока не должен использоваться как опора для сильных выводов.

Типичные причины:
- `needs_human_review`
- `unclear`
- слишком слабая доказательная база
